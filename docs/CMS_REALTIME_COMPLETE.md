## 🔄 ALUR DATA REAL-TIME

### Dari Admin ke User:

```
1. Admin login ke panel
   ↓
2. Buka CMS Beranda (Tour/Outbound)
   ↓
3. Edit konten (Slider, Testimoni, dll)
   ↓
4. Klik "Simpan Perubahan"
   ↓
5. Data tersimpan ke database (table: settings)
   ↓
6. User buka homepage
   ↓
7. Homepage fetch data dari API
   ↓
8. API ambil data dari database
   ↓
9. Konten tampil REAL-TIME di user
```

### API Endpoints:

```
GET /api/settings?key=tour_landing
→ Mengambil semua konten homepage tour

GET /api/settings?key=outbound_landing
→ Mengambil semua konten homepage outbound

POST /api/settings
→ Menyimpan perubahan dari admin
```

## 📝 NOTES PENTING

### 1. Data Disimpan di Database
Semua konten CMS disimpan di table `settings` dengan format JSON:
```sql
SELECT * FROM settings WHERE key = 'tour_landing';
SELECT * FROM settings WHERE key = 'outbound_landing';
```

### 2. Tidak Perlu Restart Server
Perubahan langsung tampil tanpa perlu restart server Next.js.

### 3. Backup Data
Sebelum ubah konten penting, backup dulu:
```bash
# Export database
mysqldump -u root -p wonw2577_tobadb > backup.sql
```

### 4. Restore Data
Jika ada kesalahan, restore dari backup:
```bash
# Import database
mysql -u root -p wonw2577_tobadb < backup.sql
```
